## 強みとして語れる経験

### 1. センサーシステムの設計・推進経験（最重要エピソード）
- 外部SIerを活用してプロジェクトを推進
- **センサーのデータ構造の決定**から担当
- **1次DB（MongoDB）の選定**：センサーデータはスキーマが変動しやすい → スキーマレスNoSQLが最適、高頻度書き込みにも強い
- **2次DBの選定**：大学仕様要件として組み込み（振り返ると1次DBのみで事足りた可能性→学びとして語れる）
- フロントエンド開発の推進
- テストの推進
- プロジェクトを成功に導いた

### 2. Python Flask → Java Spring Boot への移植経験
- 既存システムをリアーキテクチャした経験
- 技術的負債の解消として語れる
- 「御社の技術スタックに合わせて自主的に習得した」姿勢を示せる

### 3. クラウド運用経験（GAE）
- GAE（Google App Engine）= PaaS
- Heroku も同じ PaaS の概念 → 本質的な理解がある
- app.yaml でのスケーリング設定経験あり
- Docker コンテナでのデプロイ経験あり

### 4. CI/CD の理解
- Heroku の git push = CD（継続的デリバリー）の概念を実体験済み
- GitHub Actions で自動テストを組み合わせることで完全なCI/CDになると理解している

## STAR形式のエピソード（面接本番用）

**Situation（状況）**
センサーデータを収集・可視化するシステムを、外部SIerと連携して構築するプロジェクトを担当しました。

**Task（役割）**
技術選定・設計・SIerへの要件定義から、フロントエンド開発・テストまで一貫して推進する立場でした。

**Action（行動）**
1次DBはMongoDBを選定しました。理由は、センサーのデータ構造が変動しやすくスキーマレスが適していること、高頻度の書き込みに強いことです。2次DBは仕様要件として組み込みましたが、振り返ると1次DBのみで要件を満たせた可能性があり、要件を精査せずに構成を増やすリスクを学びました。

**Result（結果）**
プロジェクトを成功させました。

## 想定質問と回答の軸

**「なぜJavaで書いたのですか？」**
御社の技術スタックに合わせて、Python/FlaskのシステムをJava/Spring Bootに移植しました。Djangoという選択肢もありましたが、Spring SecurityやJPAの設計思想を実際に手を動かして理解しておきたかったため、あえてJavaを選びました。

**「クラウドはAWSではなくGCPですが？」**
GAEはHerokuと同様のPaaSです。デプロイの仕組みや自動スケーリングの概念は共通しており、本質的な設計思想は理解しています。

**「CI/CDの経験はありますか？」**
HerokuはGit pushをトリガーに自動デプロイされる仕組みで、CD的な運用は経験しています。CIについてはGitHub Actionsで自動テストを組み合わせることで完成すると認識しています。
